Test cases should be provided in JSON format.

There should be at least one test case.

Each test case consists of a description, parameter list, matcher and the expected value.

A parameter and the expected value includes a type and a value.

Possible types are "integer", "double", "string", "boolean" and its array versions (like "string array") are also supported.

Array values must be separated by "|" characters, it can also be escaped if needed.

Matcher can be either "equals" or "not equals".

 
{{'[
  {
    "description": "it should return the ...",
    "parameters": [
      {
        "type": "integer",
        "value": "2"
      },
      {
        "type": "string array",
        "value": "one|two|three|four"
      },
      ...
    ],
    "matcher": "not equals",
    "expected": {
      "type": "boolean",
      "value": "true"
    }
  },
  ...
]'}}
